# Mode: project — Portfolio Project Evaluation

Evaluate whether a project idea is worth building from a job search perspective. Score it, give a clear verdict, and produce a build plan if approved.

---

## Scoring

Rate the project idea across 6 dimensions (1–5):

| Dimension | Weight | 5 = | 1 = |
|-----------|--------|-----|-----|
| Signal for target roles | 25% | Directly demonstrates a skill required in target JDs | Unrelated to any target role |
| Uniqueness | 20% | Few or no similar public projects | Saturated — everyone has it (e.g., a basic to-do app) |
| Demo-ability | 20% | Live demo accessible in under 2 minutes | Source code only, nothing to show |
| Metric potential | 15% | Clear measurable outcome (latency, accuracy, processing time, users) | No measurable outcome possible |
| Time to MVP | 10% | Deliverable in 1–2 weeks | Requires 3+ months |
| STAR story potential | 10% | Rich problem with trade-offs and a clear result | Straightforward implementation with no interesting decisions |

---

## Verdicts

**BUILD** — Worth doing. Provide a week-by-week plan.

**SKIP** — Not worth the time at this stage. Explain why and suggest a better alternative.

**PIVOT TO [alternative]** — The core idea is good but a variation would be more impactful. Describe what to build instead.

---

## Build Plan (if verdict is BUILD)

Produce a 2-week plan:

**Week 1 — MVP with core metric**
- Day 1–2: Scope and data/API setup
- Day 3–5: Core functionality working end-to-end
- Day 7: One measurable metric captured and documented

**Week 2 — Polish and application pack**
- Day 8–10: UI or demo interface (even minimal)
- Day 11–12: README with architecture diagram and metric
- Day 13–14: Record a 2-minute demo walkthrough

---

## Application Pack Requirements

Every completed project should include:
1. **Live demo** — a working URL or recorded walkthrough (2 minutes max)
2. **One-pager** — what it does, how it works, what the metric is
3. **Brief postmortem** — what worked, what was hard, what you would do differently

These three items are what turn a project into a job search asset. Without them, the project is only visible on GitHub.

---

## Nigerian Context

For Nigerian job seekers, projects that solve local problems carry extra credibility:
- Fintech adjacent: mobile money integration, payment reconciliation, fraud detection
- Education data: WAEC/JAMB result analysis, school access mapping
- Logistics: last-mile delivery routing, street-address normalisation for Nigerian addresses
- Agriculture: crop price tracking, market data aggregation
- Public data: INEC election data visualisation, CBN exchange rate tracking

A project that uses a public Nigerian dataset signals both technical skill and local relevance — a combination that stands out in Nigerian employer evaluations.
